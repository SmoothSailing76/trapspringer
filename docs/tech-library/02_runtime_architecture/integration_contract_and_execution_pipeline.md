# Integration Contract and Execution Pipeline

## Master Loop

1. Load current procedure frame.
2. Read live state.
3. Query spatial/visibility facts.
4. Filter knowledge for recipients.
5. Present narration/prompt.
6. Accept user input.
7. Simulate other players.
8. Build declarations.
9. Validate declarations.
10. Resolve valid actions.
11. Commit state.
12. Apply knowledge/map updates.
13. Narrate outcomes.
14. Advance procedure.
15. Log everything.
16. Repeat.

## State Commit Protocol

Resolution proposes mutations. Layer 3 commits state. Layer 2 commits knowledge. Layer 9 commits map/reveal changes. Layer 10 logs. Layer 7 narrates committed public outcome.

## Ownership

Layer 1 authority; Layer 2 knowledge; Layer 3 state; Layer 4 procedure; Layer 5 validation; Layer 6 resolution; Layer 7 narration; Layer 8 party; Layer 9 space; Layer 10 audit.
